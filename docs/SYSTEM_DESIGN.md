\## 2. スプレッドシート構造



\### 2-1. Families（家族マスタ）



| 列名 | 型 | 説明 |

|------|-----|------|

| 家族ID | string | 例：F001。手動採番 |

| 宛名 | string | 例：藤本様 |

| 配信区分 | string | 自 / 直 |

| 保護者LINE\_USER\_ID | string | LINE Login で取得 |

| 登録日 | date | LINE登録日 |

| 配信有効フラグ | boolean | TRUEで配信対象 |



\### 2-2. Students（生徒マスタ）



| 列名 | 型 | 説明 |

|------|-----|------|

| 生徒ID | string | マイ活と同じID（連携用） |

| 家族ID | string | Familiesの家族IDと紐付け |

| 学年 | string | 例：中3、高1、高卒 |

| 生徒氏名 | string | 例：藤本　晴 |

| 兄弟順位 | int | 1, 2, 3（同一家族内での順位） |

| 在籍フラグ | boolean | TRUEで在籍中 |



\### 2-3. Billings（月次配信データ）



| 列名 | 型 | 説明 |

|------|-----|------|

| 配信月 | string | YYYY-MM形式 |

| 家族ID | string | |

| 合計金額 | int | 内訳の合計（自動計算） |

| 引落日 | date | 例：2026-05-26 |

| 配信本文 | text | 生成済みの配信本文 |

| 配信ステータス | string | 未配信 / テスト済 / 配信成功 / 配信失敗 |

| 配信日時 | datetime | 実配信時刻 |

| エラー詳細 | text | 失敗時のエラーメッセージ |



\### 2-4. BillingItems（明細）



| 列名 | 型 | 説明 |

|------|-----|------|

| 配信月 | string | YYYY-MM形式 |

| 家族ID | string | |

| 生徒ID | string | どの兄弟の明細か |

| 項目名 | string | 例：6月分授業料、維持費、北辰テスト6月分 |

| 金額 | int | マイナス値も可（返金等） |

| 表示順 | int | 配信本文での表示順 |



\### 2-5. FollowUps（滞納フォロー履歴）



| 列名 | 型 | 説明 |

|------|-----|------|

| 配信月 | string | YYYY-MM形式 |

| 家族ID | string | |

| 失敗理由 | string | 残高不足 / ゆうちょ未処理 |

| フォロー回数 | int | 1回目、2回目、… |

| 送信日時 | datetime | |

| 送信文面 | text | 実送信した文面 |

| 対応ステータス | string | 未対応 / 入金確認済 |

| 入金確認日 | date | |



\### 2-6. Config（設定）



| 列名 | 型 | 説明 |

|------|-----|------|

| キー | string | 設定項目名 |

| 値 | string | 設定値 |



設定例：

\- channel\_access\_token

\- admin\_line\_user\_ids（カンマ区切り、テスト受信先）

\- template\_jido（引落用テンプレ）

\- template\_chokubarai（直払用テンプレ）

\- template\_followup\_zandaka（残高不足フォロー用）

\- template\_followup\_yucho（ゆうちょ未処理フォロー用）

\## 5. 運用フロー（月次）

\[1] 引落日の約1週間前（月20日前後）

├ ふくちさんが従来のExcelファイルを当月分作成

└ 管理画面で「Excel取込」を実行

\[2] 取込後の確認

├ Billings/BillingItemsに自動書き込み

├ 配信本文プレビュー画面で全件確認

└ 必要に応じて個別修正

\[3] テスト配信

└ ふくちさんのLINEに全パターン送信（自分宛て）

\[4] 本配信

├ 件数を入力（誤配信防止）

└ 配信実行

\[5] 引落日後（月末〜翌月初）

├ 銀行から引落結果データ受領

├ 管理画面で「引落結果入力」

│   ├ 成功家族にチェック

│   └ 失敗家族に理由（残高不足/ゆうちょ未処理）を選択

└ 失敗家族にフォローLINE送信

\[6] 入金確認後

└ FollowUpsシートで「対応ステータス：入金確認済」に更新
